# Параметры конфигурации робота

Ниже приведено описание коэффициентов и пинов, которые используются прошивкой робота. Для каждого пункта указано назначение, ожидаемый порядок значений и влияние на поведение.

## Пиновая разводка
- **pml1 / pml2** — GPIO для управления левым мотором (PWM). Диапазон: номера пинов ESP32 (обычно 0–39). Неверные значения приведут к отсутствию движения соответствующего направления.
- **pmr1 / pmr2** — GPIO для управления правым мотором. Диапазон: 0–39. Ошибка в значениях даст перекрёстное или отсутствующее управление правым колесом.
- **pel1 / pel2** — пины фаз энкодера левого колеса. Диапазон: входные GPIO 0–39. Неправильный выбор исказит подсчёт импульсов, скорость и одометрию.
- **per1 / per2** — пины фаз энкодера правого колеса. Диапазон: входные GPIO 0–39. Ошибка вызовет неверное измерение скорости/пути правого колеса.
- **pled1 / pled2** — пины светодиодов индикации. Диапазон: 0–39. Используются для визуального статуса; неверные пины просто отключат индикацию.
- **pclk / psda** — пины SCL/SDA для I²C шины. Диапазон: 0–39. На них вешаются датчики/дисплеи; неправильная нумерация блокирует работу I²C.
- **pbat** — аналоговый вход измерения напряжения аккумулятора. Диапазон: ADC‑совместимые пины ESP32. Неподходящий пин приведёт к некорректному мониторингу батареи.
- **pch** — пин детектора зарядки/источника питания. Диапазон: 0–39. При ошибке индикация статуса питания будет неверной.

## Геометрия и энкодеры
- **wrad** — радиус колеса в сантиметрах. Типичные значения 2.5–4.5 см. Занижение увеличит расчётную скорость, завышение снизит оценку пройденного пути.
- **wdist** — расстояние между колёсами в сантиметрах. Обычно 15–25 см. Влияет на вычисление дуги при поворотах; неправильное значение приведёт к недоворачиванию или переворачиванию.
- **er** — число импульсов энкодера за полный оборот колеса. Порядок величины: 2000–3000 имп/об. Меньшее число делает робота «глухим» к мелким движениям, большее — «нервным», завышая оценку скорости.

## Скоростные ограничения
- **maxs** — максимально разрешённая угловая скорость колеса, рад/с. Типично 10–20 рад/с. Используется при насыщении регуляторов, ограничивает команду скорости.
- **ks** — коэффициент коррекции прямолинейности. Типичный порядок 50–150. Чем выше, тем агрессивнее подавляется разброс скоростей между колёсами при движении прямо.

## PID для угла (движение по расстоянию/углу)
- **kpa** — пропорциональный коэффициент углового регулятора. Порядок 30–150. Рост увеличивает реакцию на ошибку угла/дистанции.
- **kia** — интегральный коэффициент углового регулятора. Порядок 50–120. Ускоряет устранение статической ошибки, но при завышении даёт перерегулирование.
- **kda** — дифференциальный коэффициент углового регулятора. Порядок 1–5. Демпфирует быстрые изменения ошибки.

## PID для скорости колёс
- **kpsl / kpsr** — пропорциональные коэффициенты регуляторов скорости левого/правого колёс. Типичный порядок 20–50. Задают, насколько PWM реагирует на рассогласование фактической и заданной скорости.
- **kis** — интегральный коэффициент регулятора скорости (общий для колёс). Порядок 0–10. Убирает долгосрочную ошибку скорости.
- **kdsl / kdsr** — дифференциальные коэффициенты регуляторов скорости. Порядок 0–1. Помогают стабилизировать скорость при резких изменениях нагрузки.

## Режимы и отладка
- **ks** — коэффициент сохранения прямой траектории (дублируется в блоке скоростных ограничений для удобства). Порядок 50–150. Чем больше, тем агрессивнее компенсация дрейфа.
- **debug** — флаг вывода отладочных сообщений (0 — выкл, 1 — вкл). Не влияет на движение, но полезен при настройке.

### Практическая настройка
1. Начните с геометрии: уточните `wrad`, `wdist`, `er` по механике и паспорту энкодера.
2. Проверьте пины моторов и энкодеров: при необходимости переставьте значения `pml*`, `pmr*`, `pel*`, `per*` под свою плату.
3. Настройте PID угла (`kpa`, `kia`, `kda`) для устойчивых поворотов и точного хода по расстоянию.
4. Подберите PID скорости (`kpsl`, `kpsr`, `kis`, `kdsl`, `kdsr`) так, чтобы колёса плавно набирали скорость без рывков.
5. Отрегулируйте `ks` и `maxs` под баланс между прямолинейностью и максимальной скоростью.
6. При необходимости добавьте индикацию и телеметрию через `pled*`, `pclk/psda`, `pbat`, `pch`.
